2. 脚本开发基础

转至元数据起始

一、组件开发思想

图1组件开发

UnityGame 是由1或n个scene（场景）组成，每个scene（场景）拥有许多GameObject（游戏对象）。

游戏对象本身不做任何事，他们需要通过Component(组件)才能完成功能（比如空间变换）。

组件有Unity引擎内置可以直接使用的系统组件（比如 提供空间变换功能的 Transform 组件），也提供开发者接口以备自己编写特定的组件。

开发者需要创建script（脚本）（官方有 C#、 Boo、 JavaScript 三种选择，本文选用C#，C#基本语法请读者自己先熟悉。），

脚本默认继承于MonoBehaviour（Mono行为类，unity引擎的一个基类，当然脚本不使用组件属性和行为也可以不继承）。

实际操作

1.创建脚本：菜单栏 Assets -> Create -> C# Script 创建脚本名为CodeTrainning

2.创建游戏对象：菜单栏 GameObject ->Create Empty 创建一个空游戏对象 命名为 MyGameObject

3.在Hierarchiy面板 选择游戏对象 MyGameObject， 并在Inspector 视图中单击对象属性下方的 Add Component 按钮，在弹出的菜单中依次打开 Scripts -> CodeTrainning。

这样就将CodeTrainning脚本绑定到MyGameObject游戏对象，Project 视图中将 CodeTrainning脚本直接拖动到 Inspector 视图中的 MyGameObject对象上来完成脚本绑定操作。


二、脚本事件执行顺序

脚本本身只是一个组件，是无法脱离游戏对象独立运行的，它必须绑定到游戏对象上才会生效。而脚本一旦添加到游戏对象直到游戏对象销毁过程，

系统顺序派发特定的事件并调用脚本的事件响应函数。脚本整个生命周期一般我们常用的重点事件执行顺序如下图：

图2 事件执行顺序（由于是入门教程，此图只绘制一般常用的事件，详细事件可以见https://docs.unity3d.com/Manual/ExecutionOrder.html）

MonoBehaviour （Mono行为类）提供系统脚本事件响应方法以备开发者重写。为更好编写自定义组件，就要先了解这些事件。

三、脚本和组件交互

组件为游戏对象提供功能，在脚本我们希望访问游戏对象自身其他组件，使用某些功能，那么我们得需要了解在脚本怎样与组件交互。

  • 获取组件

常用组件可以通过gameObject的内置变量访问

除了以上常用组件，其他组件可以通过以下方法获取：

GetComponent ：获取组件

GetComponents：获取组件列表

GetComponentlnChildren：获取对象或对象子物体上的组件

GetComponentsInChildren：获取对象或对象子物体上的组件列表

 这几个函数都比较耗时，因此一般不要在Update等逐帧调用函数中使用，应该在外面获取组件的引用。

特殊访问方式：Unity提供另外一种非常便捷访问组件或游戏对象的方法，通过声明访问权限为 Public 的变量然后将要访问的组件或者对象赋值给该变量，

就可以在脚本中通过变量访问组件或对象了

四、操作场景中的游戏对象

我们在上文已经了解如何在脚本获取游戏对象本身的组件，有时候我们还需使用访问其他游戏对象，并与之交互，比如隐藏、显示某个游戏，或者访问某个游戏对象的组件，使用某些功能，这个

时候我们就得来了解如何访问场景中的游戏对象。

  • 获取游戏对象
  1. 通过名称来查找：使用函数GameObject.Find
  2. 通过标签来查找：GameObject.FindWithTag & GameObject.FindGameObjectsWithTag

GameObject.Find、GameObject.FindWithTag和GameObject.FindGameObjectsWithTag也是比较耗时的参数，因此不建议在Update等逐帧调用函数中使用，而是在初始化中查找一次后保存在变量中。

  • 隐藏/显示游戏对象

五、常用组件和类

为了提高效率,Unity提供一些有用的组件和类，我们需要了解才能更快捷的开发脚本。

  • Tranform（空间变换组件）
    1. 父子层次
           childTransfrom.parent = parentTransfrom
    2. 空间变换

      • 位置

        position世界坐标系中的位置
        localPosition父对象局部坐标系中的位置

      • 旋转

        rotation世界坐标系中以四元数表示的旋转
        localRotation父对象局部坐标系中以四元数表示的旋转


  • Coroutine（协程）

Coroutine即协同程序或协程，可以和主程序并行运行，和多线程类似，但是在任一制定时刻只会有一个协同程序在运行，别的协同程序则会挂起。可以用来实现让一段程序等待一段时间后继续运行的效果。

协同程序(Coroutine)的相关函数如下：

      • StartCoroutine：启动一个协同程序
      • StopCoroutine：停止一个协同程序
      • StopAllCoroutine：终止所有协同程序
      • WaitForSeconds：等待若干秒
      • WaitForFixedUpdate：等待直到下一次FiexdUpdate调用

  • Time 类 具体API参考官方文档 https://docs.unity3d.com/ScriptReference/Time.html
  • Random类 具体API参考官方文档 https://docs.unity3d.com/ScriptReference/Random.html
  • Mathf类 具体API参考官方文档https://docs.unity3d.com/ScriptReference/Mathf.html

    如有遇到需要查UnityEngine的某个类API查：https://docs.unity3d.com/ScriptReference/
    需要查某个UnityEngine引擎功能以及用法可以查：https://docs.unity3d.com/Manual/index.html
